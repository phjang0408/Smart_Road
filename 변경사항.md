> 자세한 사항은 [`개선필요부분.md`](개선필요부분.md) 참고

**필수 사항**
1. DataManager 완성 및 저장 검증 - SaveResult 클래스 도입, 상세 로깅 추가
2. 역주행 버튼 기능 수정 - ForceWrongWay() 메서드로 강제 역주행 구현
3. 타이머 동기화 - DateTime 기반으로 변경하여 배속 동작 정확화
4. CSV 저장 시 타임스탬프 오류 - DateTime.Now 대신 기록 시점 타임스탬프 사용
5. 신호 제어 로직 이중화 - TrafficController의 신호 판단을 UI에 반영, MainWindow 자체 타이머 제거
6. TrafficLight 참조 공유로 데이터 오염 - record 타입 또는 Clone() 메서드로 값 복사
7. IDisposable 미준수 및 메모리 누수 - 명시적 Dispose() 호출, 이벤트 핸들러 해제

**개선하면 좋겠는 부분**

8. 계층 분리 - TrafficPolicyManager 중간 계층 추가, UI와 로직 분리
9. 렌더링 성능 저하 - O(N²) 충돌 감지를 방향별 분류로 최적화
10. 예외 처리 강화 - 로깅 시스템 도입, 사용자 친화적 에러 메시지 추가
11. 단위 테스트 추가 - NUnit/xUnit 테스트 프로젝트 추가, DataGenerator/TrafficController 테스트
13. SensorData에 시뮬레이션 시각 정보 누락 - SensorData에 SimHour 추가, DataGenerator에서 생성 시 포함
14. MainWindow의 매직 넘버 및 Brush 중복 생성 - 좌표 상수 정의

**UI/UX 부분**
- 보행자 UI에 추가 (횡단보도 검토)
- 역주행 차량 UI에 안 뜨는 버그

- Road Safety Score 가시성 보완

## 주요 개선사항

### 1. 일일 시뮬레이션 모드로 수정 (Daily Simulation)
- **DataGenerator.cs**
  - `_simHour`, `_isDayRainy`, `_dayMinTemp`, `_dayMaxTemp` 상태 변수 추가
  - `Initialize()`: 상태만 초기화 (타이머 미시작)
  - `StartDay()`: 명시적 타이머 시작 메서드 추가
  - `DayCompleted` 이벤트 추가
  - `OnTimerTick()`: 명명된 이벤트 핸들러 (메모리 누수 방지)
  - `UpdateSensorData()`: `_simHour >= 24` 체크 추가 (조기 종료 방지)

### 2. 현실적 날씨 시뮬레이션 - 데이터를 하루 단위로 맞춰 보완
- **DataGenerator.cs**
  - `StartNewDay()`: 하루 단위 날씨 결정 (30% 확률 비)
  - `GenerateTemperature()`: 코사인 곡선 (최저 04:00, 최고 14:00)
  - `GenerateRainfall()`: 일일 날씨 상태 기반 결정
  - `GetRoadCondition()`: 영하 자동 결빙 (강수 조건 제거)

### 3. UI 개선
- **MainWindow.xaml**
  - 도로 위 화살표: 크기 20→36, 색상 파란색
  - 신호등: 노란불 제거, 90도 회전, 위치 조정
  - 블랙아이스 이펙트: `rectBlackIceOverlay` 추가
  - 역주행 차량: `rectWrongWayCar` 빨간색 표시
  - 그래프 섹션: Weather & Environment 위쪽 배치

- **MainWindow.xaml.cs**
  - 블랙아이스 이펙트: Opacity 제어 (0 또는 0.25)
  - 역주행 차량: 시각적 경고 표시
  - `RenderTimer_Tick`: `_isDayRunning` 체크 추가 (시뮬레이션 종료 후 차량 정지)
  - Road Safety Score추가
  - 시뮬레이션 시작 버튼 추가 (소요시간은 하단 9번)
### 4. 자동 저장 시스템
- **DataManager.cs**
  - `AutoSaveDay()`: `Desktop/SmartRoad_Data/{yyyy-MM-dd}/` 폴더 자동 생성
  - `ResetForNewDay()`: 매일 데이터 초기화

### 5. 그래프 시스템 추가
하루 기준, 기온과 강수량을 토대로 그래프를 나타냄
- **MainWindow.xaml.cs**
  - `_temperatureHistory`, `_rainfallHistory`: 24시간 데이터 저장
  - `DrawGraph()`:
    - x축 : 시간, y축 : 데이터값
    - 밝은 파란색, 밝은 초록색
    - ActualWidth 사용으로 반응형 레이아웃
  - `UpdateGraphStats()`: 최고/최저 기온, 누적강수 표시

### 6. 상태 초기화
- **MainWindow.xaml.cs**
  - `ResetRoadState()`: 하루 완료 후 도로 상황 초기화
    - 신호등 모두 빨강
    - 도로 상태 → 건조
    - 상태값 초기화 (차량 유지)

### 7. 버그 수정
- 온도 곡선 방향 오류 (sine→cosine)
- 타이머 이벤트 핸들러 메모리 누수
- 초기 0시 데이터 스킵
- 하루 끝난 후 데이터 계속 생성
- 하루 시뮬레이션 이후, 차량이 계속 움직이는 문제
- 그래프 표시 안 됨
