## 브랜치 전략

```
main
  └── develop
        ├── feature/data-generator    (장지욱)
        ├── feature/score-logic       (최민석)
        ├── feature/ui-dashboard      (송성조)
        └── feature/test-save         (손강표)
```

- 각자 feature 브랜치에서 작업
- develop에 PR(Pull Request) → 팀원 1명 이상 리뷰 후 merge
- main은 항상 실행 가능한 상태 유지
- 기능 완성 시 develop → main merge

## 커밋 메시지 규칙(임시)
임시로 지정했습니다. 확정되면 따로 카톡에 올리겠습니다.
```
feat: 기능 추가
fix: 버그 수정
ui: UI 관련 변경
test: 테스트 추가 또는 수정
docs: 문서 수정
refactor: 코드 리팩토링
chore : 환경설정이나 파일이동(.gitignore나 README.md)
style : 코드 형식이나 가독성 수정할 때 (세미콜론 누락, 들여쓰기 교정 등)
rename : 파일 혹은 폴더명 변경

```

예시:
```
feat: 기온-노면 상태 종속 관계 구현
fix: 결빙 조건 임계값 수정
ui: 신호등 황색 갱신 연동
test: 블랙아이스 시나리오 테스트 추가
```
