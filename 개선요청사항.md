# 개선요청사항

## 배속 시 카운트다운 동기화 문제

**어떤 문제인지**
시뮬레이션 배속을 올려도 신호등 카운트다운 속도가 변하지 않음.

**왜 문제인가?**
`TrafficController`가 `DateTime.Now`(현실 시간)를 기준으로 초를 계산하기 때문임. 배속(2x, 4x)이 적용되어도 신호등은 현실 시간 1배속으로 동작함.

**어디 부분인지**

- 파일: `TrafficController.cs` 라인 176-180 (신호 전환 시간 계산 로직)

**개선을 어떻게 하면 좋을지**
현실 시간 의존성을 제거하고, UI 물리 엔진에서 전달하는 deltaTime(배속이 반영된 흐른 시간)을 받아 카운트다운을 감소시키도록 수정 필요.

```csharp
public void PassTime(double deltaTime)
{
    // 데이터 수신 여부와 상관없이 매 프레임 시간 차감
    _currentPhaseTotalTime -= deltaTime;
}
```

## 데이터 동기화 시간 문제

**어떤 문제인지**
신호 전환 시점에서 "신호 전환 대기 중..."상태가 너무 길게 지속됨

**왜 문제인가?**
UI는 30FPS로 즉시 0초에 도달하지만, `TrafficController`는 `DataGenerator`로부터 다음 센서 데이터(Tick)를 수신받을 때까지 신호 전환 연산을 수행하지 않음.
즉, 데이터 갱신 주기(Tick)와 UI 렌더링 주기 간의 비동기성으로 인해 발생하는 대기 시간임.

**어디 부분인지**

- 파일: `TrafficController.cs` 라인 88-92, 110-115 (`OnSensorDataUpdated`, `ExecuteSignalControl`)

**개선을 어떻게 하면 좋을지**
`TrafficController`가 외부 센서 데이터를 기다리지 않고, UI에서 전달받는 PassTime(deltaTime)을 사용하여 독자적으로 남은 시간을 감소시키고 상태를 변경하도록 로직 분리 필요.

```csharp
public void PassTime(double deltaTime)
{
    _currentPhaseTotalTime -= deltaTime;

    // 시간이 0 이하가 되면 외부 데이터 수신을 기다리지 않고 즉시 신호 전환 로직 실행
    if (_currentPhaseTotalTime <= 0)
    {
        ExecuteSignalControl();
    }
}
```
